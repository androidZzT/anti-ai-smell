# renhua —— 中文去 AI 味

AI 写中文有一套固定的「味儿」：它爱把英文概念硬翻成生硬的词（`门控`、`横切`、`一等公民`），爱造母语者不用的新词（`切法`、`硬货`），爱给平淡的事套戏剧化形容（`理直气壮的谎`），还爱用宏大官腔（`赋能`、`涌现`、`不仅仅…而是…`）。这些词单看没错，但堆在一起，读者一眼就知道「这是 AI 写的」。

这个 skill 做一件事：**把这些词标出来，换成人话。**

它和句式类去 AI 味工具（如 humanizer-zh，或 Wikipedia「Signs of AI writing」那套）互补——那些管句子结构，这个管**中文特有的词汇口味**。

## 什么时候用

- 写完一篇中文文章 / 帖子 / 文案 / README / 代码注释，交付前去 AI 味。
- 用户说「这句 AI 味太重」「读着不像人话」「太生硬 / 翻译腔」「这个词没人这么说」。
- 翻译或转述英文技术内容后（翻译腔高发区）。

## 怎么用

### 1. 扫一遍

```bash
python3 lint.py 你的稿子.md
```

输出按类别列出命中的词，每个词给出人话替代和命中次数。例如：

```
⚠ article.md  命中 6 处（hard 1）
  翻译腔
    门控 (gate / gating) ×3  →  把关 / 卡一道 / 网关  // ML 的 gating…那种场合可用
    收敛 (converge) ×1  →  慢慢定下来 / 归到一处  // 数学、迭代算法里精确，别乱替
  宏大 / 官腔
    赋能 ×1  →  帮 / 让……能……
  句式
    破折号滥用 ×4  →  能用逗号或句号就别用破折号
```

### 2. 按语境判断，别机械替换

词表分两档：

- **hard（宏大官腔、套话）**：几乎总是该改。`赋能` `涌现`（非术语义）`不仅仅` `值得注意的是` `此外`——直接删或换。lint 命中 hard 会非零退出。
- **soft（生造、翻译腔、戏剧化、带味字、句式）**：看语境。有些词在专业场合有精确含义，别一刀切：
  - `门控`：机器学习的 gating（按输入选专家）、电路的门，是精确专有词，那种上下文里保留。
  - `收敛`：数学、迭代算法里是术语，别乱替。
  - `幂等` `编排` 这类**正常术语不在词表里**——圈内人正常用，不算 AI 味。

判断标准就一句：**这话，一个母语者平时会不会这么说。** 会，就留；不会，换成人话那列。

### 3. 换完复扫

```bash
python3 lint.py 你的稿子.md
# 无 hard 级命中。soft 级按语境自行判断。
```

## 看整张词表

```bash
python3 lint.py --list
```

分五类：生造词、翻译腔、戏剧化、带味的字、宏大官腔 + 套话/连接词，外加几条 AI 味句式（破折号滥用、修辞性设问、否定式排比等）。词表在 `data/ai-flavor-words.json`，可自己加词。

## 硬规则

- **不机械替换**。lint 只标位置和建议，改不改、怎么改，人（或调用它的 agent）按语境定。尤其 soft 级和有 `note` 的词。
- **正常术语不算 AI 味**。幂等、编排、缓存、幂零……圈内人正常用的词不收进来。这个 skill 针对的是「生造 / 翻译腔 / 戏剧化 / 官腔」，不是「所有专业词」。
- **和句式工具配合**。这个管词，句式（三段式、否定排比、金句、-ing 分析）交给 humanizer 那类工具，或看 `data` 里的 `syntax_patterns` 自查。

## 加词

`data/ai-flavor-words.json` 里每条是 `{"w": 词, "en": 英文源, "human": [人话...], "note": 备注}`。发现新的 AI 味词，往对应类别里加，或提 PR。判断能不能收：母语者平时不这么说，但 AI 老写它。
